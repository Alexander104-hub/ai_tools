====== Синтакс форматирования ======

[[doku>DokuWiki]] поддерживает простой язык разметки, который пытается сделать файлы данных максимально читабельными. Эта страница содержит весь возможный синтаксис, который вы можете использовать при редактировании страниц. Просто посмотрите исходный код этой страницы, нажав «Редактировать эту страницу». Если вы хотите что-то попробовать, просто воспользуйтесь страницей [[playground:playground|playground]]. Более простая разметка также легко доступна через [[doku>панель инструментов|быстрые кнопки]].

===== Основное форматирование текста =====

«ДокуВики» поддерживает **жирный**, //курсив//, __подчеркнутый__ и «моноширинный» тексты. Конечно, вы можете **__//''объединить''//__** все это.

  «ДокуВики» поддерживает **жирный**, //курсив//, __подчеркнутый__ и «моноширинный» тексты.
  Конечно, вы можете **__//''объединить''//__** все это.

Вы также можете использовать <sub>нижний индекс</sub> и <sup>надстрочный индекс</sup>.

  Вы также можете использовать <sub>нижний индекс</sub> и <sup>надстрочный индекс</sup>.

Вы также можете пометить что-либо как <del>удаленное</del>.

  Вы также можете пометить что-либо как <del>удаленное</del>.

**Абзацы** создаются из пустых строк. Если вы хотите **перевести новую строку** без абзаца, вы можете использовать две обратную косую черту, за которой следует пробел или конец строки.

Это текст с переносами строк\\ Обратите внимание, что
две обратные косые черты распознаются только в конце строки\\
или за ним следует \\ пробел \\ это происходит без него.

  Это текст с переносами строк\\ Обратите внимание, что
  две обратные косые черты распознаются только в конце строки\\
  или за ним следует \\ пробел \\ это происходит без него.

Вы должны использовать принудительный перевод строки только в том случае, если это действительно необходимо.

===== Ссылки =====

«ДокуВики» поддерживает несколько способов создания ссылок.

==== Внешние ссылки ====

Внешние ссылки распознаются автоматически: http://www.google.com или просто www.google.com. Вы также можете установить текст ссылки: [[http://www.google.com|Эта ссылка указывает на Google] ]. Адреса электронной почты, подобные этому: <andi@splitbrain.org>, также распознаются.

  «ДокуВики» поддерживает несколько способов создания ссылок. Внешние ссылки распознаются
  автоматически: http://www.google.com или просто www.google.com – вы можете установить
  а также текст ссылки: [[http://www.google.com|Эта ссылка указывает на Google]]. Электронная почта
  адреса, подобные этому: <andi@splitbrain.org>, также распознаются.

==== Внутренние ссылки ====

Внутренние ссылки создаются с помощью квадратных скобок. Вы можете просто указать [[имя страницы]] или использовать дополнительный [[имя страницы|текст ссылки]].

  Внутренние ссылки создаются с помощью квадратных скобок. Вы можете просто дать
  [[имя_страницы]] или используйте дополнительный [[имя_страницы|текст ссылки]].

[[doku>pagename|Имена страниц Wiki]] автоматически преобразуются в нижний регистр, специальные символы не допускаются.

Вы можете использовать [[some:namespaces]], используя двоеточие в имени страницы.

  Вы можете использовать [[some:namespaces]], используя двоеточие в имени страницы.

Подробную информацию о пространствах имен см. в [[doku>пространства имен]].

Также возможна ссылка на конкретный раздел. Просто добавьте имя раздела за символом решетки, известным из HTML. Это ссылка на [[syntax#internal|этот раздел]].

  Это ссылка на [[syntax#internal|этот раздел]].

Примечания:

* Ссылки на [[синтаксис|существующие страницы]] отображаются в другом стиле, чем [[несуществующие]].
  * DokuWiki не использует [[wp>CamelCase]] для автоматического создания ссылок по умолчанию, но это поведение можно включить в файле [[doku>config]]. Подсказка: если «ДокуВики» является ссылкой, то она включена.
  * При изменении заголовка раздела меняется и его закладка. Так что не полагайтесь слишком сильно на ссылки разделов.

==== Интервики ====

DokuWiki поддерживает ссылки [[doku>Интервики]]. Это быстрые ссылки на другие вики. Например, это ссылка на страницу Википедии о Вики: [[wp>Wiki]].

  DokuWiki поддерживает ссылки [[doku>Интервики]]. Это быстрые ссылки на другие вики.
  Например, это ссылка на страницу Википедии о Вики: [[wp>Wiki]].

==== Общие ресурсы Windows ====

Общие ресурсы Windows, такие как [[\\server\share|this]] также распознаются. Обратите внимание, что это имеет смысл только в однородной группе пользователей, такой как корпоративная [[wp>Интранет]].

  Общие ресурсы Windows, такие как [[\\server\share|this]], также распознаются.

Примечания:

  * По соображениям безопасности прямой просмотр общих ресурсов Windows по умолчанию работает только в Microsoft Internet Explorer (и только в «локальной зоне»).
  * Для Mozilla и Firefox его можно включить с помощью другого обходного пути, упомянутого в [[http://kb.mozillazine.org/Links_to_local_pages_do_not_work|База знаний Mozilla]]. Однако предупреждение JavaScript о попытке открыть общий ресурс Windows по-прежнему будет отображаться. Чтобы удалить это предупреждение (для всех пользователей), поместите следующую строку в файл conf/lang/en/lang.php (более подробную информацию можно найти в [[doku>localization#changing_some_localized_texts_and_strings_in_your_installation|localization]]): <code - conf/lang/en/lang.php>
<?php
/**
* Настройка файла английского языка
 * Копируйте только те строки, которые необходимо изменить.
 */
$lang['js']['nosmblinks'] = '';
</code>

Обратите внимание: форматирование изображения — единственный синтаксис форматирования, принятый в именах ссылок.

Поддерживается весь синтаксис [[#images_and_other_files|image]] и [[#links|link]] (включая изменение размера изображения, внутренние и внешние изображения, URL-адреса и интервики-ссылки).

==== Поддерживаемые форматы мультимедиа ====

«ДокуВики» может напрямую встраивать следующие медиаформаты.

| Image | ''gif'', ''jpg'', ''png''  |
| Video | ''webm'', ''ogv'', ''mp4'' |
| Audio | ''ogg'', ''mp3'', ''wav''  |
| Flash | ''swf''                    |

Если вы укажете имя файла, которое не является поддерживаемым форматом мультимедиа, вместо этого оно будет отображаться в виде ссылки.

Добавляя «?linkonly», вы предоставляете ссылку на медиафайл, не отображая ее в строке.

  {{wiki:dokuwiki-128.png?linkonly}}

{{wiki:dokuwiki-128.png?linkonly}} Это просто ссылка на изображение.

==== Резервные форматы ====

К сожалению, не все браузеры понимают все форматы видео и аудио. Чтобы решить эту проблему, вы можете загрузить файл в разных форматах для максимальной совместимости с браузером.

Например, рассмотрим встроенное видео в формате mp4:

  {{video.mp4|A funny video}}

Когда вы загружаете файлы «video.webm» и «video.ogv» рядом с указанным «video.mp4», DokuWiki автоматически добавит их в качестве альтернатив, чтобы ваш браузер понимал один из трех файлов. .

Кроме того, «ДокуВики» поддерживает изображение «постера», которое будет показано перед началом видео. Это изображение должно иметь то же имя файла, что и видео, и быть файлом формата jpg или png. В приведенном выше примере подойдет файл «video.jpg».

===== Списки =====

«Докувики» поддерживает упорядоченные и неупорядоченные списки. Чтобы создать элемент списка, отступите текст на два пробела и используйте «*» для неупорядоченных списков или «-» для упорядоченных.

  *Это список
  *Второй пункт
    * У вас могут быть разные уровни
  * Еще один предмет

  - Тот же список, но упорядоченный
  - Еще один предмет
    - Просто используйте отступ для более глубоких уровней
  - Вот и все

<code>
  *Это список
  *Второй пункт
    * У вас могут быть разные уровни
  * Еще один предмет

  - Тот же список, но упорядоченный
  - Еще один предмет
    - Просто используйте отступ для более глубоких уровней
  - Вот и все
</code>

Также ознакомьтесь с [[doku>faq:lists|FAQ по элементам списка]].

===== Текстовые преобразования =====

«ДокуВики» может конвертировать определенные заранее определенные символы или строки в изображения, другой текст или HTML.

Преобразование текста в изображение в основном выполняется для смайлов. Преобразование текста в HTML используется для замены типографики, но его также можно настроить на использование другого HTML.

==== Преобразование текста в изображение ====

«ДокуВики» преобразует часто используемые [[wp>смайлы]] в их графические эквиваленты. Эти [[doku>Смайлы]] и другие изображения можно настраивать и расширять. Вот обзор смайлов, включенных в «ДокуВики»:

  * 8-) %%  8-)  %%
  * 8-O %%  8-O  %%
  * :-( %%  :-(  %%
  * :-) %%  :-)  %%
  * =)  %%  =)   %%
  * :-/ %%  :-/  %%
  * :-\ %%  :-\  %%
  * :-? %%  :-?  %%
  * :-D %%  :-D  %%
  * :-P %%  :-P  %%
  * :-O %%  :-O  %%
  * :-X %%  :-X  %%
  * :-| %%  :-|  %%
  * ;-) %%  ;-)  %%
  * ^_^ %%  ^_^  %%
  * m(  %%  m(   %%
  * :?: %%  :?:  %%
  * :!: %%  :!:  %%
  * LOL %%  LOL  %%
  * FIXME %%  FIXME %%
  * DELETEME %% DELETEME %%

==== Преобразование текста в HTML ====

Типографика: [[DokuWiki]] может преобразовывать простые текстовые символы в их типографически правильные объекты. Вот пример узнаваемых персонажей.

-> <- <-> => <= <=> >> << -- --- 640x480 (c) (tm) (r)
«Он думал: «Это мужской мир»…»

<code>
-> <- <-> => <= <=> >> << -- --- 640x480 (c) (tm) (r)
«Он думал: «Это мужской мир»…»
</code>

То же самое можно сделать и для создания любого HTML-кода, его просто нужно добавить в [[doku>entities|файл шаблона]].

Есть три исключения, которые не взяты из этого файла шаблона: объект умножения (640x480), «одинарные» и «двойные кавычки». Их можно отключить с помощью опции [[doku>config:typography|config]].

===== Цитирование =====

Иногда вам нужно пометить какой-то текст, чтобы показать, что это ответ или комментарий. Вы можете использовать следующий синтаксис:

<code>
Я думаю, нам следует это сделать

> Нет, нам не следует

>> Ну, я говорю, что мы должны

> Правда?

>> Да!

>>> Тогда давай сделаем это!
</code>

Я думаю, нам следует это сделать

> Нет, нам не следует

>> Ну, я говорю, что мы должны

> Правда?

>> Да!

>>> Тогда давай сделаем это!

===== Таблицы =====

«ДокуВики» поддерживает простой синтаксис для создания таблиц.

^ Заголовок 1 ^ Заголовок 2 ^ Заголовок 3 ^
| Строка 1 Столбец 1 | Строка 1 Колонка 2 | Строка 1 Колонка 3 |
| Строка 2 Колонка 1 | немного colspan (обратите внимание на двойную трубку) ||
| Строка 3 Колонка 1 | Строка 3 Колонка 2 | Строка 3 Столбец 3 |

Строки таблицы должны начинаться и заканчиваться символом «|» для обычных строк или символом «^» для заголовков.

  ^ Заголовок 1 ^ Заголовок 2 ^ Заголовок 3 ^
  | Строка 1 Столбец 1 | Строка 1 Колонка 2 | Строка 1 Колонка 3 |
  | Строка 2 Колонка 1 | немного colspan (обратите внимание на двойную трубку) ||
  | Строка 3 Колонка 1 | Строка 3 Колонка 2 | Строка 3 Столбец 3 |

Чтобы соединить ячейки по горизонтали, просто сделайте следующую ячейку полностью пустой, как показано выше. Убедитесь, что у вас всегда одинаковое количество разделителей ячеек!

Вертикальные заголовки таблиц также возможны.

| ^ Рубрика 1 ^ Рубрика 2 ^
^ Заголовок 3 | Строка 1 Колонка 2 | Строка 1 Колонка 3 |
^ Заголовок 4 | на этот раз без colspan | |
^ Заголовок 5 | Строка 2 Столбец 2 | Строка 2 Колонка 3 |

Как видите, форматирование решает разделитель ячеек перед ячейкой:

  | ^ Рубрика 1 ^ Рубрика 2 ^
  ^ Заголовок 3 | Строка 1 Колонка 2 | Строка 1 Колонка 3 |
  ^ Заголовок 4 | на этот раз без colspan | |
  ^ Заголовок 5 | Строка 2 Столбец 2 | Строка 2 Колонка 3 |

Вы можете создать диапазоны строк (вертикально соединенные ячейки), добавив «%%:::%%» в ячейки ниже той, к которой они должны подключаться.

^ Заголовок 1 ^ Заголовок 2 ^ Заголовок 3 ^
| Строка 1 Столбец 1 | эта ячейка располагается вертикально | Строка 1 Колонка 3 |
| Строка 2 Колонка 1 | ::: | Строка 2 Колонка 3 |
| Строка 3 Колонка 1 | ::: | Строка 2 Колонка 3 |

Помимо синтаксиса rowspan, эти ячейки не должны содержать ничего другого.

  ^ Заголовок 1 ^ Заголовок 2 ^ Заголовок 3 ^
  | Строка 1 Столбец 1 | эта ячейка располагается вертикально | Строка 1 Колонка 3 |
  | Строка 2 Колонка 1 | ::: | Строка 2 Колонка 3 |
  | Строка 3 Колонка 1 | ::: | Строка 2 Колонка 3 |

Вы также можете выровнять содержимое таблицы. Просто добавьте как минимум два пробела на противоположном конце текста: добавьте два пробела слева для выравнивания по правому краю, два пробела справа для выравнивания по левому краю и два пробела как минимум на обоих концах для выравнивания текста по центру.

^ Таблица с выравниванием ^^^
| верно | центр |слева |
|слева | верно | центр |
| хххххххххх | хххххххххх | хххххххххх |

Вот как это выглядит в исходнике:

  ^ Таблица с выравниванием ^^^
  | верно | центр |слева |
  |слева | верно | центр |
  | хххххххххх | хххххххххх | хххххххххх |

Примечание. Вертикальное выравнивание не поддерживается.

===== Без форматирования =====

Если вам нужно отображать текст точно так, как он набран (без форматирования), заключите область тегами «%%<nowiki>%%» или, что еще проще, двойными знаками процента «»<nowiki>%%< /новики>''.

<новики>
Это какой-то текст, который содержит такие адреса: http://www.splitbrain.org и **форматирование**, но с ним ничего не делается.
</новики>
То же самое справедливо и для %%//__this__ text// со смайлом ;-)%%.

  <новики>
  Это какой-то текст, который содержит такие адреса: http://www.splitbrain.org и **форматирование**, но с ним ничего не делается.
  </новики>
  То же самое справедливо и для %%//__this__ text// со смайлом ;-)%%.

==== Подсветка синтаксиса ====

==== Загружаемые блоки кода ====

Когда вы используете синтаксис «%%<code>%%» или «%%<file>%%», как указано выше, вы также можете захотеть сделать показанный код доступным для загрузки. Вы можете сделать это, указав имя файла после кода языка, например:

<code>
<file php myexample.php>
<?php echo "hello world!"; ?>
</file>
</code>

<file php myexample.php>
<?php echo "hello world!"; ?>
</file>

Если вам не нужна подсветка, но вам нужен загружаемый файл, укажите тире (''-'') в качестве кода языка: ''%%<code - myfile.foo>%%''.

===== Агрегация каналов RSS/ATOM =====
[[DokuWiki]] может интегрировать данные из внешних XML-каналов. Для анализа XML-каналов используется [[http://simplepie.org/|SimplePie]]. Все форматы, понятные SimplePie, также можно использовать в «ДокуВики». Вы можете повлиять на рендеринг с помощью нескольких дополнительных параметров, разделенных пробелами:

^ Параметр ^ Описание ^
| любое число | будет использоваться в качестве максимального количества отображаемых элементов, по умолчанию — 8 |
| реверс | сначала отображать последние элементы в ленте |
| автор | показать имена авторов |
| дата | показать даты элемента |
| описание| покажите описание товара. Все HTML-теги будут удалены |
| носорт | не сортировать элементы в ленте |
| //n//[дхм] | период обновления, где d=дни, h=часы, m=минуты. (например, 12 часов = 12 часов). |

Период обновления по умолчанию составляет 4 часа. Любое значение ниже 10 минут будет считаться 10 минутами. [[wiki:DokuWiki]] обычно пытается предоставить кэшированную версию страницы; очевидно, это неприемлемо, если страница содержит динамический внешний контент. Параметр указывает [[wiki:DokuWiki]] повторно отобразить страницу, если с момента последнего отображения страницы прошло больше //периода обновления//.

По умолчанию лента будет отсортирована по дате: сначала самые новые элементы. Вы можете отсортировать его по принципу «сначала самые старые», используя параметр «reverse», или отобразить канал как есть с помощью «nosort».

**Пример:**

  {{rss>http://slashdot.org/index.rss 5 автор дата 1 час }}

{{rss>http://slashdot.org/index.rss 5 автор дата 1 час }}


===== Макросы управления =====

Некоторый синтаксис влияет на то, как «ДокуВики» отображает страницу, не создавая никакого вывода самостоятельно. Доступны следующие управляющие макросы:

^ Макрос ^ Описание |
| %%~~NOTOC~~%% | Если этот макрос найден на странице, оглавление создаваться не будет |
| %%~~NOCACHE~~%% | «ДокуВики» по умолчанию кэширует весь вывод. Иногда это может быть нежелательно (например, когда используется приведенный выше синтаксис %%<php>%%), добавление этого макроса заставит DokuWiki перерисовывать страницу при каждом вызове |
